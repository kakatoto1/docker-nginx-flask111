Project outline

Project Root Folder <- Root of the project

docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku

Dockerfile <-File used to create an image to run in a container that runs the program

Heroku.yml <- The heroku.yml file is a manifest you can use to define your Heroku app.

Pytest.ini <- File used to hold pytest configuration.

Readme.md <- generate the html summary to see at the bottom of projects.

Setup.py <- The setup script is the central of all activity in building, distributing, and installing modules using the Distutils.

Workflows Foldee <- .github Folder

Dev.yml <- The configuration file for development workflow.

Prod.yml <- The configuration file for production workflow.

Controllers Folder <- App Folder

Controller.py <- The Python controller runs a Python script when a sensor triggers the controller.

*_controller.py <- The controller file runs a Python script when a sensor triggers the * controller.

Templates Folder <- Simple pages folder <- App Folder

*.html <- * .html file is a HyperText Markup Language file format used as the basis of web page “*”.

_init_.py <- Definition of show(page) and set up 404 returning code. Any other module can add some additional attributes to it.

Css Folder <- Static Folder <- App Folder

Style.css <- A classless stylesheet for HTML documents.

Images Folder <- Static Folder <- App Folder

*.jepg <- Stored images.

Js Folder <- Static Folder <- App Folder

Scripts.js <- JS (JavaScript) file contains JavaScript code for execution on web pages.

Templates Folder <- App Folder

_init_.py <- Definition of create_app, any other module can add some additional attributes to it.

Run.py <- Import create_app, any other module can add some additional attributes to it.
